# 그립 시스템 재설계 — 다물체 동역학 기반 (하드코딩 폐기)

> 사용자 지시: "다물체 동역학을 하면서 잡게 해야해. 하드코딩으로 할게 아니라 제대로."
> "꼼꼼하게 시스템으로 검증해서 다른 제품이 들어와도 어렵지 않게."

## 문제 (현재 방식의 근본 결함)

현재 `grip_ops.GRIP_PRESETS` + `runner._grip_phone`의 `arm_offset`은 **특정 폰 하나에 맞춰
스윕으로 찾은 하드코딩 각도/오프셋**이다. 결함:

- 폰 크기·비율·위치가 바뀌면 전부 깨진다 (엄지 rz=-1.8, arm_offset=-20 등은 70×150×8 전용).
- 접촉을 물리적으로 풀지 않는다 → 엄지가 폰을 관통(사용자 지적 "또 통과했잖아").
- `resolve_finger_contact`의 이분탐색도 하드코딩 프리셋을 스케일할 뿐, 새 형상에 일반화 안 됨.

## 목표 시스템

**입력**: 임의의 폰/제품 외곽 STL (크기·비율·위치 무관).
**출력**: 손이 물리적으로 그 제품을 쥔 포즈 — 관통 없이 표면에 접촉, 각 손가락이 형상에 맞게 감쌈.
**핵심 원칙**: 각도를 코드에 박지 않는다. **기하 분석 + 접촉 동역학**으로 매번 계산.

## 아키텍처 (모듈 `grip_solver.py`)

### 1단계 — 그립 프레임 자동 분석 (`analyze_grip_frame`)
폰 bbox/주축에서 그립 좌표계를 계산:
- **clamp축** = 가장 얇은 치수 (두께) → 손가락·엄지가 이 축으로 마주 문다.
- **wrap 모서리** = 손가락이 감쌀 긴 모서리 (손 폭에 맞는 것).
- **손 배치** = 손바닥이 뒷면, 손가락이 wrap 모서리를 넘게. 폰 크기에 비례해 위치·회전 산출.
- 하드코딩 오프셋(-20,-9,-14) 제거 → 폰 치수의 함수로.

### 2단계 — 다물체 동역학 손가락 닫힘 (핵심)
**방식 A (우선): rigid body 시뮬레이션.**
- 각 지골(phalanx)을 rigid body, 관절을 힌지 constraint(1-DOF 굴곡).
- 각 관절에 모터/스프링으로 닫는 토크. 폰은 passive 충돌체.
- Bullet 스텝 → 손가락이 폰에 닿아 멈추고 감싼다.
- 난제: 얇은 폰 터널링, 진동 → substep↑, 충돌 마진, 각속도 감쇠, 속도 상한으로 해결.
- rigid body 결과(지골 방향) → armature 본 회전으로 매핑 → 스킨 메쉬 포즈.

**방식 B (폴백): 접촉 구동 순차 관절 닫힘 (기하 질의).**
- rigid body가 스킨 손에서 불안정하면: 각 손가락을 근위→원위 순으로,
  각 지골을 그 지골 메쉬가 폰에 접촉할 때까지 굴곡 증가 후 정지.
- 이것도 "물체 간 접촉으로 멈춤"이라 물리적으로 근거 있고 일반화됨(형상 질의).
- 방식 A보다 결정론적·안정적. 동역학의 정적 근사.

### 3단계 — 검증 하네스 (`test_grip_solver.py`)
**여러 제품으로 체계 검증** (일반화 증명):
- 합성 폰: 얇은(6mm)/두꺼운(12mm), 작은(60×120)/큰(80×160), 정사각(100×100).
- 실제 배터리 (`/data/battery_study`, 75×72×2mm 극薄).
- 각각: (1)관통 ≤ tol, (2)모든 손가락 접촉, (3)엄지 대향, (4)함몰 전파 유효.
- 정투영 3면도 렌더로 육안 + 정량 지표 동시 검증.

## 검증 기준 (통과 조건)
- [ ] 관통: 어떤 제품이든 max penetration ≤ 0.5mm.
- [ ] 접촉: 손가락 4개 + 엄지 모두 표면 접촉(min_gap ≈ 0).
- [ ] 대향: 엄지가 손가락 반대 면/모서리 (clamp).
- [ ] 일반화: 5종 이상 제품에서 재튜닝 없이 성공.
- [ ] 함몰: 접촉부에서 폰 함몰 → 모핑 전파 → 재해석 .k.

## 진행 로그
- 프로토타입: rigid body 손가락 힌지+모터 headless 구동 확인. 터널링/진동 발생.
- **터널링/진동 해결 (검증됨)**: substeps 80 + solver_iter 60 + 지골 linear/angular_damping 0.9
  + 약한 모터(target_vel 0.8, max_impulse 0.3) + 충돌 마진 0.04. 단일 손가락이 폰 표면에서
  진동 없이 안정 안착(tip_z 마지막5프레임 [7.33×5] 완전 고정). 힌지축 = empty를 Y로 90°회전해
  X축(굴곡) 정렬. **다물체 동역학 그립이 headless에서 안정적으로 동작함이 증명됨.**
- 다음: 물리 결과 → 본 매핑. 각 지골 proxy를 armature 본에 정렬 생성 → 시뮬 안착 →
  pose_bone.matrix = proxy.matrix_world로 전사 → 스킨 메쉬가 물리 그립으로 포즈.
- 그 다음: 5손가락+엄지 전체, 그립 프레임 자동 분석, 5종 제품 검증 하네스.

### 채택 방식 결정
방식 A(rigid body 시뮬)로 확정 — 안정 안착 검증됨. 사용자 요구("다물체 동역학")에 정확히 부합.
proxy→본 매트릭스 전사로 스킨 매핑. 방식 B(기하 순차닫힘)는 불필요(A가 안정적).
